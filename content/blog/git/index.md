В этом посте я хочу поделиться тем, как Git помог мне управлять версиями сайта, а GitHub Actions — автоматически публиковать его.

## Git: основные команды

В процессе работы я использовала базовые команды:

- git init — инициализация репозитория.
- git clone — копирование удалённого репозитория.
- git add — добавление изменений в индекс.
- git commit -m "сообщение" — создание коммита.
- git push — отправка изменений на GitHub.
- git pull — получение обновлений.

Также пришлось решать конфликты, когда я вносила правки одновременно локально и на GitHub. Их разрешение потребовало ручного редактирования файлов.

## CI/CD: GitHub Actions

Для автоматической сборки и публикации сайта я использовала GitHub Actions. В моём репозитории создан файл .github/workflows/gh-pages.yml, который при каждом push в ветку main:

1. Устанавливает Hugo Extended и Node.js.
2. Устанавливает зависимости (TailwindCSS).
3. Собирает сайт командой hugo --minify.
4. Загружает артефакт и деплоит его на GitHub Pages.

Это и есть пример непрерывной интеграции и непрерывного развертывания (CI/CD): при каждом изменении кода автоматически выполняется сборка и публикация. Мне не нужно вручную загружать файлы на хостинг.

## Заключение

Git и CI/CD — мощные инструменты, которые экономят время и предотвращают ошибки. Теперь я понимаю, как они работают, и смогу применять их в будущих проектах.
