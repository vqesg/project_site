На этой неделе я работала над первым этапом проекта — размещением заготовки сайта на GitHub Pages. Сайт генерируется с помощью Hugo (Extended версия) на основе темы Academic.

## Что было сделано

1. Установка Hugo и Git на Fedora в VirtualBox. Сначала пыталась установить через стандартные репозитории, но для работы нужна extended версия, поэтому скачала бинарник с GitHub.
2. Клонирование шаблона starter-hugo-academic и инициализация модулей Hugo.
3. Создание репозитория на GitHub и настройка удалённого доступа.
4. Настройка конфигурации — указала baseURL в config/_default/hugo.yaml.
5. Создание GitHub Actions workflow для автоматической публикации на GitHub Pages.
6. Удаление ссылок на Unsplash из демо-постов, чтобы избежать ошибок сборки.
7. Установка TailwindCSS для стилей (локально и через npm в workflow).
8. Исправление ошибок в YAML-файлах, где случайно использовались HTML-комментарии.

## Что получилось

Сайт успешно собран и опубликован. Теперь он доступен в интернете. В следующей части я добавлю информацию о себе и создам посты.

## Что было сложным

- Поначалу не могла понять, почему workflow падает — оказалось, не хватало установки TailwindCSS.
- HTML-комментарии в YAML-блоке вызвали ошибки при сборке.
- Пришлось разобраться с personal access token для git push.

В целом, работа с Hugo и GitHub Pages оказалась интересной, а автоматическая публикация через Actions сильно упрощает жизнь.
